AppIQ

Overview

AppIQ and AppDynamics work together to assist the user to map each application to a recommended APIC end point and allows the user a visual guide of the running state of those configurations.

Requirements

The following are the AppIQ requirements:

  • Permission level: read

  • Permissions:

    • tenant-connectivity-mgmt

    • tenant-connectivity-util

    • tenant-connectivity-l2

    • tenant-epg

    • tenant-ext-connectivity-util

    • tenant-network-profile

    • tenant-protocol-l2

    • tenant-protocol-ops

    • tenant-protocol-util

    • vmm-ep

    • vmm-connectivity

  • App version 1.0

  • Minimum required controller version: 2.2(1n)

  • On premise AppDynamics controller version: 4.5.1 and above.

  • AppIQ integration requires AppDynamics monitored applications to be running on this ACI fabric and the mapping of End points will only show up if the End points are present under the given Tenant.

Login

Once the AppIQ has been enabled, click on your AppDynamics tenant and the AppIQ tab on the far left of the menu bar to login. Enter the following information:

Property Description

AppDynamics Controller

The AppDynamics controller IP address or Hostname.

AppDynamics Controller Port

The running port identifier.

AppDynamics User

The controller user name.

AppDynamics Account

The tenant user account name.

AppDynamics User Password

The tenant user account password.

List of Applications

Displays the list of applications from AppDynamics and recommends the appropriate APIC end points to deploy them on.

Property Description

Application Name

The AppDynamics tenant application name.

Health

Displays the application statistics.

  • Status—The application current health state. The state is expressed from normal to critical depending on the current health state of the application running on AppDynamics.

  • Details—Lists the Apic end points and which EPGs, and AppD applications they are associated with and their health states.

  • Mapping—Allows the user to add or remove AppDynamic applications to the recommended APIC end points.

  • View—Displays a tree format view of all the tenant associated end points. Each branch is color coded from the top at Application Profile>EPG>EP>Node. The health can be viewed by the color that encircles each point.

    Note

    Hover over each point to view the end point statistics.

Mapping

Add or remove AppDynamic applications to the recommended APIC end points.

View

Use the View display to see detailed information on all the tenant EPGs within AppDynamics:

NumberDescription

1 - 3

The AppDynamics health status. The health status color is shown as the circled color surrounding each point on the tree graph. The status can be:

  • Red - critical

  • Yellow - warning

  • Green - normal

4

The APIC EPG, EP, and node name.

5

The AppDynamics EPG or EP tier name.

6

Hover over each point to see details on how the end points connect to the APIC and their contract information.

7

Expresses the number of ACI EPs in the EPG not mapped to any AppDynamics application.

CaveatsDescription

Total ACI Tenants supported

Recommended for having AppDynamics EPs mapped with up to 3 ACI Tenants.

AppDynamics Controller

The AppIQ app can only point to 1 AppD Controller at a time.

AppDynamics User Privileges

AppD configured user on AppIQ needs to have privileges to access AppD public APIs.

AppDynamics Communication Access

The app uses https access by default. If the AppDynamics controller communicates over http, add http://AppDynamicsControllerIP/ in the login page of AppIQ.

ACI Communication Access

If the https access to APIC is disabled, the app will try to communicate via http. To ensure HTTPS communication, enable HTTPS access in ACI.
